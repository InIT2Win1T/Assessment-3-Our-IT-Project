Plans and Progress

 

Our team is InIt2Win1T, and our project idea began when the five of us came together during our Introduction to Information Technology course for the purpose of completing our second assignment for the course, which was also our first group-based assignment.

 

We entered a server on Discord for us to use for informal discussion relating to the assignment as well as some occasional jokes or discussion that was not directly related to the assignment. For formal assignment meetings, we conversed together on Microsoft Teams.

 

During our first meeting for Assignment 2, we delegated various roles for the assignment amongst ourselves. We decided that we were open to acquiring an extra member if anyone in the course was left without a group, but that we were happy with the five members as it was. We lost one of our members, Alarn, who dropped out of the course shortly after that first meeting. We then immediately acquired Josh, who was in a different class to the rest of us, and who was able to pick up from where we began and took on the roles that had been delegated to Alarn.

 

In the first meeting, we all discussed what our project ideas were that we explained for our work on Assignment 1. After having shared our ideas, the question was put forward as to which one of these to work on for Assignment 2, or whether to choose another new idea. We collectively decided that Justin’s idea was the most appropriate, as we believed it was the most feasible idea. We decided upon our team’s name, which is InIt2Win1T.

 

Justin’s idea for Assignment 1, which was what we chose for Assignment 2, was to create an app and website for AFL Fantasy players that would provide detailed statistical information of AFL players that will enable AFL Fantasy players to make more informed choices as to how they play. This would work by acquiring and storing data of AFL players using Microsoft Excel and providing users with information in the form of player cards, line graphs, and box plots, and various other useful information. Justin was surprised such a tool did not already exist already, and that current websites that are pre-existing do not give detailed-enough information.

 

During our work on Assignment 2, we had to discuss our ideal jobs, and compare these to current trends and job demands in the IT industry. Josh primarily worked on this task. Shang created our team profile, which discussed our varying personality types and gave us group roles. Justin interviewed an individual who works in an IT role, who have us some helpful insights into his role and what it entails. Simon explained the project idea and a range of technologies needed for it. Nick researched some areas of IT such as cybersecurity, machine learning, and cloud computing. Simon added some research on small computing devices as well. Nick created the website for our project ideas to be reported and displayed, with Shang doing much of the managing of this with GitHub. We were happy with the work we all contributed and believed we worked well together. Every member believed that every other member made a great deal of contributions to the assignment, and we were very glad to submit what we had created. When we had our meetings on Teams, we would create an agenda for the meeting and record minutes of what was discussed. Nick frequently demonstrated strong leaderships skills, his knowledge and experience in IT contributed well to this also.

 

After submitting Assignment 2 we continued this same project idea in Assignment 3. This time, we were to work on a “whiteboarding” phase for this project idea and create some tangible artifacts for the project. As mentioned, the project was to be to create both an app and a website but we decided it would be more feasible to work on the website rather than the app. This was because we felt it would be easier to develop a website and that a website didn’t need to be platform-specific, and have both an Android and iOS version created. An app would be a more long term-goal however, depending on how successful the website would be.

 

We have been keeping busy in the past six weeks, creating artifacts for our project as well as completing many other elements. We are currently at a point where we have defined technologies needed- which would firstly include a virtual server on Amazon Lightsail, which would use cPanel to host and manage various services used to develop the website. We would also use security services provided by Amazon Web Services. Adobe Dreamweaver would be used for the front-end of the website. Microsoft Excel would be used to store scraped data from relevant web sources on AFL Football players and to generate graphs and other data-analysis elements for the website. We will need to use Apify to perform the web scrape.

 

Justin has worked with Excel to create mockup charts that display information on AFL players. He has also used some formulars to calculate averages and standard deviations. Simon has also made some line graphs of AFL player’s fantasy points, discovering that when a player did not play in a certain game it would record as “0” on the graph. These could be left out by replacing these values with “#N/A”. Simon tried to perform a web scrape with Excel but was not successful using Excel’s tools for this purpose. Justin and Simon and created some player cards- Simon’s being a simplified one, possibly for the free version of our website. Nick is creating the website to report our project idea as well as discussing testing and creating a project timeline. Shang has created some job descriptions for jobs that we would need if this website was to be developed in actuality as well as discussing the group processes. Josh has laid out the aims and goals for this project. In addition to these tasks mentioned, everyone also performed several other tasks and helped each other out with the various elements of our project work.

 

We are at a point now where we have a more solid idea of how the website will be hosted, developed and deployed, we understand risks associated with creating this website, and have identified employment positions we would need to commence the project. We have also set up some tangible artifacts for this project. We are also creating a presentation with PowerPoint which will be converted to a video which will cover the basic of our project- what it is, our competitors, technology involved, how we aim to create revenue, and our SWOT analysis.

 

After the completion of Assignment 3, we would like to seeking out the technologies we need, such as acquired cloud services and the virtual server and software needed to develop the website. We then intend to begin early development phases, which will include creating back-end code, creating a system to use Apify to perform web scrapes and store the information in Excel. During this time we would also like to create some mockups for our website design as well as for user interface elements, particularly player cards.

 

Overall, the journey has been a learning experience for us all and we have enjoyed working together and creating this project. Should the opportunity arise, we would be happy to take this project onwards in its next steps.
