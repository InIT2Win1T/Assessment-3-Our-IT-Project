Tools and Technology

 

In terms of physical equipment, our team would require computers such as desktops or laptops that are connected to the internet. It will be possible for each of us to be in distant locations and to work collaboratively on this project online. This will be made possible through the use of online virtual servers, cloud computing and GitHub which will be used for software version control.

 

Web development tools, such as Adobe Dreamweaver or Wix, would be essential for the early stages of our project (Drake and Turner, 2021). Both Dreamweaver and Wix require a subscription to plan to use. This is to create the aesthetics of the page and to create much of the front-end elements of the website. Dreamweaver allows more customisation in terms of site design as well as offering both a GUI and coding editor (as per Drake and Turner, 2021) and so it is our tool of choice. GitHub would be required to keep track of the developmental process, particularly in terms of creating software versions for the back-end website features. Nickolas has a considerable degree of experience with web development.

 

Statistics on AFL players, especially their “fantasy points” will be acquired from relevant AFL sources using Microsoft Excel by the use of web scraping and stored in these Excel spreadsheets. Justin and myself (Simon) have a degree of experience using Excel and would be able to write formulas. Excel requires a subscription to use. We will scrape data from a page similar to the following:

 

https://www.footywire.com/afl/footy/pg-hawthorn-hawks--ben-mcevoy

 

The website would require hosting. A cloud service on Amazon Web Services, Microsoft Azure or Google Cloud would cater for this. Using Amazon Lightsail, which also is available with a subscription service and is scalable, we would use a remote virtual server running Linux to build our website with (Hall,2019). Information on Lightsail can be found here:

 

https://aws.amazon.com/lightsail/?did=ft_card&trk=ft_card

 

On this virtual server, we will use cPanel, which is a Linux-based web hosting control panel that allows the management all the services in one single space (Domantas, 2021). Due to the range of tools cPanel provides a range of useful tools to create the back-end of our website. These include Ruby tools, PHP and PEAR packages IP blockers for security, shopping cart tools, SQL tools, GitHub tools and more. One thing our site will need, is advertisement APIs for the free version of our website. cPanel can provide this too. For a cPanel-based website, we can manage it with WHS as well. (Irfan, 2021)

 

We would need to develop Application Programming Interfaces (APIs) for the purpose of making the web app functional with end users. These would mediate between the users and the data they are inputting, to return the data they are looking for that based on what is stored within our Excel spreadsheets. (Mulesoft, 2021)

 

We have tried web scraping from relevant websites using Excel, without any worthwhile success. Although Javascript or Python scripts can be created to achieve web scraping (Breuss, 2021), we believe Apify would be a much-needed tool to enable us to scrape the data we need. To schedule our web scrapes to be done each Sunday, we could use Cron to achieve this (Pair Knowledge Base, 2020). Information on Apify can be found here:

 

https://apify.com/

 

Another important technological aspect of our site is the security, as it will require users to enter data and therefore this will mean data will need to be protected. cPanel does offer a number of security tools and Amazon Web Services has Guard Duty and Inspector. Guard Duty is designed to cautiously monitor AWS accounts in order to detect malicious activity. Inspector is used for checking for vulnerabilities and features automated security checking during development and deployment as well as giving security teams an ability to look into the web app’s security testing. Amazon Web Application Firewall and Amazon Shield would also be good to protect from SQL injection attacks and assist with traffic mitigation. These services also have subscription services.

 

Information on Inspector can be found here: https://aws.amazon.com/inspector/?did=ft_card&trk=ft_card

 

Information on Guard Duty can be found here: https://aws.amazon.com/guardduty/?did=ft_card&trk=ft_card

 

Information on Web Application Firewall can be found here:

 

https://aws.amazon.com/waf/
